# AI Workbench 微信公众号发布配置指南

## 1. 功能简介

AI Workbench 可以将当前 Obsidian Markdown 笔记转换并提交到微信公众号草稿箱。

插件提供两种接入方式：

1. **官方 API**：插件直接调用微信公众号接口创建草稿。
2. **Webhook**：插件把文章发送给自建或第三方中转服务，由中转服务创建公众号草稿。

官方 API 适合拥有公众号开发权限、网络出口 IP 相对稳定的用户。Webhook 适合需要集中管理凭据、固定服务器 IP、扩展排版或接入其他发布系统的用户。

插件只创建草稿，不会自动群发或公开发布。草稿创建后，应进入公众号后台检查排版、预览内容并手动发布。

---

## 2. 发布能力

微信公众号官方 API 模式支持：

- 创建单篇公众号草稿
- 设置标题
- 设置作者
- 设置摘要
- 转换 Markdown 正文
- 上传封面图片
- 上传正文中的本地图片
- 保留远程图片地址
- 返回微信草稿 ID

当前不支持：

- 自动群发或公开发布
- 将视频作为公众号草稿正文发布
- 多图文草稿
- 完整复刻 Obsidian 主题样式
- 复杂 Markdown 表格、代码高亮和自定义组件转换
- 自动同步公众号后台对草稿进行的修改

---

## 3. 使用前准备

### 3.1 公众号要求

使用官方 API 前，请确认：

- 可以登录[微信公众平台](https://mp.weixin.qq.com/)
- 公众号后台可以查看 AppID
- 可以生成或重置 AppSecret
- 公众号具备素材管理和草稿箱相关接口权限
- 可以配置接口调用 IP 白名单

不同账号类型、认证状态和运营地区拥有的接口权限可能不同。请以公众号后台“接口权限”页面显示的结果为准。

### 3.2 网络要求

运行 Obsidian 的电脑需要能够访问：

- `https://api.weixin.qq.com`
- 微信公众平台相关素材接口

如果使用代理、VPN、公司网关或动态家庭宽带，需要特别注意公网出口 IP 是否变化。

## 4. 获取公众号凭据

### 4.1 获取 AppID

1. 登录微信公众平台。
2. 打开 `设置与开发`。
3. 进入 `开发接口管理` 或 `基本配置`。
4. 找到开发者 ID，即 AppID。
5. 复制并妥善保存。

不要把微信号、公众号原始 ID 或二维码信息填写为 AppID。

### 4.2 获取 AppSecret

在同一页面找到开发者密码 AppSecret。

如果后台不再显示原密钥，需要按公众号后台要求完成管理员验证并重置 AppSecret。

注意：

- AppSecret 重置后，旧密钥立即失效。
- AppSecret 相当于公众号接口密码，不应通过聊天、截图或公开仓库传播。
- 修改 AppSecret 后，需要同步更新插件设置。

### 4.3 配置 IP 白名单

微信公众号获取 `access_token` 时可能检查请求来源 IP。

1. 在公众号后台进入开发基本配置。
2. 找到 IP 白名单。
3. 查询当前网络的公网出口 IP。
4. 将该公网 IPv4 地址添加到白名单。
5. 保存配置。

不要填写：

- `127.0.0.1`
- `localhost`
- `192.168.x.x`
- `10.x.x.x`
- 其他局域网地址

如果使用代理或 VPN，应填写微信接口实际看到的出口 IP。家庭宽带、移动热点和部分企业网络的公网 IP 可能变化。此前可用但突然认证失败时，应首先检查 IP 白名单。

## 5. 在 AI Workbench 中配置官方 API

### 5.1 打开发布设置

在 Obsidian 中依次打开：

`设置 → 第三方插件 → AI Workbench → 发布平台`

找到“微信公众号”。

### 5.2 基础设置

1. 打开微信公众号右侧的启用开关。
2. 将接入方式选择为“官方 API”。
3. 点击“配置”。
4. 确认“启用平台”已经打开。
5. 根据需要开启“工作台默认选中”。

“工作台默认选中”打开后，每次打开 AI 工作台，微信公众号会默认处于选中状态。

### 5.3 填写字段

| 字段 | 必填 | 说明 |
|---|---:|---|
| AppID | 是 | 微信公众平台中的开发者 ID |
| AppSecret | 是 | 微信公众平台中的开发者密码 |
| 默认作者 | 否 | 创建草稿时写入的默认作者名称 |

填写完成后点击“保存”。

### 5.4 请求超时

发布平台区域提供“发布请求超时”设置，默认值为 60 秒。

建议：

- 普通纯文字文章：60 秒
- 包含多张图片：120 秒
- 网络速度较慢：120～180 秒

超时时间过短可能导致图片仍在上传时任务被中止。超时时间过长则会延迟网络错误反馈。

## 6. 测试连接

在微信公众号设置行点击“测试连接”。

测试连接会：

1. 读取已保存的 AppID 和 AppSecret。
2. 调用微信 `access_token` 接口。
3. 验证基础认证和网络连接。
4. 显示连接结果。

测试连接不会：

- 读取或发送当前笔记
- 上传图片
- 创建测试草稿
- 公开发布内容

显示“连接正常”表示插件能够获取 `access_token`。素材上传和草稿创建权限仍需要通过首次实际发布进行完整验证。

## 7. 准备 Obsidian 文章

### 7.1 推荐文章结构

```markdown
---
title: 示例文章标题
summary: 这是文章摘要
tags:
  - AI
  - 效率工具
---

# 示例文章标题

这是文章正文。

## 第一部分

正文支持**粗体**、*斜体*、链接和列表。

![[images/cover.png]]
```

### 7.2 标题读取顺序

插件按以下优先级获取文章标题：

1. YAML Frontmatter 中的 `title`
2. 正文第一个一级标题
3. Markdown 文件名

发布编辑器打开后，可以手动修改标题。

### 7.3 图片要求

插件可以识别：

```markdown
![[images/example.png]]
```

以及：

```markdown
![图片说明](images/example.png)
```

远程图片可以使用 HTTPS 地址：

```markdown
![图片说明](https://example.com/example.png)
```

发布前请确认本地图片文件仍然存在，并且可以在 Obsidian 中正常显示。

## 8. 创建公众号草稿

### 8.1 从 AI 工作台发布

1. 打开需要发布的 Markdown 笔记。
2. 打开 AI Workbench 侧边栏。
3. 找到“发布到草稿箱”。
4. 勾选“微信公众号”。
5. 点击“编辑并发布”。
6. 检查统一内容。
7. 根据需要切换到公众号平台标签修改专属内容。
8. 选择封面。
9. 检查正文图片。
10. 提交发布任务。

### 8.2 从命令面板发布

打开 Obsidian 命令面板，执行：

`发布当前笔记到平台草稿箱`

如果已设置默认平台，插件会使用默认平台；否则会使用所有已启用并完成配置的平台。

### 8.3 发布编辑器字段

| 字段 | 说明 |
|---|---|
| 标题 | 公众号文章标题，不能为空 |
| 正文 | 从 Markdown 笔记提取，可在发布前修改 |
| 摘要 | 写入公众号草稿摘要 |
| 封面 | 公众号官方草稿必填 |
| 正文图片 | 随文章上传并替换为微信图片地址 |
| 标签 | 供多平台统一内容使用，公众号官方草稿当前不会直接写入标签 |

### 8.4 发布结果

成功后，结果区域会显示公众号发布成功和草稿 ID。

然后进入微信公众平台：

`内容与互动 → 草稿箱`

找到新建草稿，检查排版并预览。

## 9. 官方 API 发布流程

插件创建草稿时依次执行：

1. 使用 AppID 和 AppSecret 获取 `access_token`。
2. 将封面上传为微信永久缩略图素材。
3. 上传正文中的本地图片。
4. 使用微信返回的图片 URL 替换 Markdown 图片引用。
5. 将 Markdown 正文转换为基础 HTML。
6. 调用微信新增草稿接口。
7. 保存并显示微信返回的草稿 ID。

任何步骤失败都会终止本次公众号草稿创建，不会自动改用 Webhook。

---

## 10. Markdown 转换规则

当前版本支持以下基础转换：

| Markdown | 公众号 HTML |
|---|---|
| `# 标题` | HTML 标题 |
| 普通段落 | `<p>` 段落 |
| `- 列表` | 无序列表 |
| `1. 列表` | 有序列表 |
| `**粗体**` | 粗体 |
| `*斜体*` | 斜体 |
| `[文字](URL)` | 超链接 |
| 本地图片 | 上传后生成 `<img>` |

以下内容可能需要在公众号后台重新调整：

- Markdown 表格
- 嵌套列表
- 代码块和语法高亮
- 脚注
- Mermaid 图表
- Obsidian Callout
- 数学公式
- 自定义 HTML
- Obsidian 主题 CSS

建议将插件生成的内容视为“可编辑草稿”，正式群发前务必在公众号后台预览。

## 11. 封面与正文图片

### 11.1 封面

微信公众号官方 API 模式必须选择封面。

封面会通过微信永久素材接口上传，并将返回的素材 ID 用作草稿封面。

如果没有封面，插件会阻止提交并提示：

`微信公众号草稿必须选择封面`

### 11.2 正文图片

本地正文图片会被读取为二进制数据并上传到微信正文图片接口。上传成功后，插件会使用微信返回的 URL 替换正文中的原始图片路径。

远程图片不会重新上传，插件会保留其 URL。为了提高公众号后台的稳定性，关键图片建议使用本地文件并由插件上传。

### 11.3 视频

微信公众号官方模式当前不支持视频草稿。视频字段主要用于其他发布平台或 Webhook。

## 12. Webhook 模式

### 12.1 适用场景

以下情况适合使用 Webhook：

- 当前电脑公网 IP 经常变化
- 不希望在 Obsidian 本地保存 AppSecret
- 需要在服务器上统一管理多个公众号
- 需要使用专业公众号排版引擎
- 需要审核、定时任务或发布队列
- 已有内容管理系统或自动化平台

Webhook 不是微信官方提供的现成 URL。你需要自行部署中转服务，或者使用兼容 AI Workbench 协议的服务。

### 12.2 插件设置

在微信公众号设置中：

1. 启用平台。
2. 接入方式选择“Webhook”。
3. 点击“配置”。
4. 填写 Webhook URL。
5. 如果文章包含本地图片，填写媒体上传 URL。
6. 选择认证方式。
7. 根据需要填写签名密钥。
8. 保存并测试连接。

### 12.3 Webhook 字段

| 字段 | 必填条件 | 说明 |
|---|---:|---|
| Webhook URL | 必填 | 接收测试和创建草稿请求 |
| 媒体上传 URL | 包含本地媒体时必填 | 接收封面、正文图片或视频 |
| 认证方式 | 必填 | 无认证、Bearer Token、自定义请求头 |
| Bearer Token | 选择 Bearer 时必填 | 写入 `Authorization` 请求头 |
| 自定义请求头 JSON | 选择自定义请求头时必填 | 例如 `{"X-API-Key":"value"}` |
| 签名密钥 | 否 | 用于 HMAC-SHA256 请求签名 |

公网服务地址必须使用 HTTPS。本地开发时只有 `localhost` 和 `127.0.0.1` 可以使用 HTTP。

### 12.4 连接测试请求

插件向 Webhook URL 发送：

```json
{
  "version": "1",
  "type": "connection-test",
  "platform": "wechat"
}
```

服务返回 HTTP 2xx 即可表示基础连接成功。也可以返回：

```json
{
  "success": true
}
```

连接测试不会携带文章内容。

### 12.5 媒体上传请求

插件使用 `multipart/form-data` 请求媒体上传 URL，字段包括：

| 字段 | 说明 |
|---|---|
| `file` | 媒体文件 |
| `taskId` | 发布任务 ID |
| `platform` | 固定为 `wechat` |
| `idempotencyKey` | 幂等键 |

媒体服务应返回：

```json
{
  "success": true,
  "mediaRef": "media-reference-id"
}
```

### 12.6 创建草稿请求

插件向 Webhook URL 发送 JSON：

```json
{
  "version": "1",
  "taskId": "publish-task-id",
  "idempotencyKey": "idempotency-key",
  "platform": "wechat",
  "target": "draft",
  "content": {
    "title": "文章标题",
    "bodyMarkdown": "Markdown 正文",
    "summary": "文章摘要",
    "tags": ["AI", "效率工具"]
  },
  "media": {
    "cover": {
      "mediaRef": "cover-reference",
      "name": "cover.png",
      "kind": "image"
    },
    "images": [],
    "video": null
  },
  "metadata": {
    "sourcePath": "文章路径.md"
  }
}
```

中转服务创建草稿后应返回：

```json
{
  "success": true,
  "draftId": "wechat-draft-id",
  "managementUrl": "https://example.com/drafts/wechat-draft-id"
}
```

其中：

- `success` 必须为 `true`
- `draftId` 必须是字符串
- `managementUrl` 可选

### 12.7 认证与签名

Bearer 模式会添加：

```http
Authorization: Bearer YOUR_TOKEN
```

每个请求都会添加：

```http
Idempotency-Key: generated-key
```

配置签名密钥后，还会添加：

```http
X-AI-Workbench-Timestamp: 1710000000
X-AI-Workbench-Signature: signature
```

签名算法：

```text
HMAC-SHA256(signingSecret, timestamp + "." + rawBody)
```

服务端应验证时间戳、原始请求体和签名，并设置合理的时间窗口以防止重放攻击。

## 13. 数据安全

平台凭据保存在 Obsidian 本地插件数据中，默认没有额外加密。

请遵守以下安全建议：

- 不要分享插件 `data.json`
- 不要将真实 AppSecret 提交到 Git
- 不要在截图或录屏中暴露密钥
- 定期轮换 AppSecret 和 Webhook Token
- Webhook 公网服务必须使用 HTTPS
- 服务端应验证认证信息和签名
- 为中转服务设置最小必要权限
- 怀疑密钥泄露时立即重置

如果 Obsidian 仓库会通过云盘或 Git 同步，应确认插件数据是否也会被同步。

## 14. 常见问题与故障排查

### 14.1 工作台中公众号不可选

检查：

1. 是否启用了微信公众号。
2. 接入方式是否正确。
3. 官方模式是否填写 AppID 和 AppSecret。
4. Webhook 模式是否填写有效 URL。
5. 修改配置后是否点击保存。

### 14.2 认证失败

典型提示：

`微信公众号认证失败，请检查 AppID 和 AppSecret`

排查顺序：

1. 检查 AppID 是否复制完整。
2. 检查 AppSecret 是否属于同一个公众号。
3. 检查 AppSecret 是否被重置。
4. 检查当前公网 IP 是否在白名单中。
5. 暂时关闭 VPN 或确认代理出口 IP。
6. 检查公众号接口权限和账号状态。

### 14.3 无法连接微信公众号接口

检查：

- 当前网络是否正常
- 是否能访问 `api.weixin.qq.com`
- 防火墙是否拦截 Obsidian
- 代理软件是否允许 Obsidian 联网
- DNS 是否正常
- 请求超时是否过短

### 14.4 封面上传失败

检查：

- 是否已选择封面
- 封面文件是否存在
- Obsidian 是否可以读取图片
- 图片格式、尺寸和文件大小是否符合微信当前要求
- 公众号是否具备永久素材接口权限

### 14.5 正文图片上传失败

检查：

- 图片路径是否正确
- 图片文件是否被移动或删除
- 图片是否损坏
- 图片是否过大
- 图片格式是否受支持
- 网络是否在上传过程中中断

### 14.6 草稿创建失败

如果认证和图片上传都成功，但创建草稿失败，可能原因包括：

- 公众号没有草稿箱接口权限
- 标题或正文不符合微信限制
- 封面素材 ID 无效
- 正文 HTML 包含不受支持的内容
- 微信接口临时异常
- 公众号达到素材、草稿或接口调用限制

### 14.7 插件显示成功但后台找不到

检查：

- 登录的公众号是否与 AppID 对应
- 是否在正确公众号的草稿箱中查找
- 是否存在多个公众号管理员账号
- 发布结果中的草稿 ID 是否有效

### 14.8 Webhook 测试失败

检查：

- Webhook URL 是否为 HTTPS
- 本地 HTTP 地址是否使用 `localhost` 或 `127.0.0.1`
- 服务是否接受 POST 请求
- Bearer Token 或自定义请求头是否正确
- 防火墙是否开放端口
- 服务是否在规定时间内返回 HTTP 2xx

### 14.9 Webhook 媒体上传失败

检查：

- 是否填写媒体上传 URL
- 服务是否接受 `multipart/form-data`
- 响应是否包含 `success: true`
- 响应是否包含字符串类型的 `mediaRef`
- 上传接口是否允许当前文件大小

## 15. 推荐配置方案

### 15.1 个人用户

- 使用官方 API
- 配置 AppID、AppSecret 和 IP 白名单
- 超时设置为 120 秒
- 每次发布后进入公众号后台检查排版

### 15.2 团队或多公众号

- 使用服务器 Webhook
- 在服务器集中保管公众号凭据
- 使用 HTTPS、Bearer Token 和 HMAC 签名
- 为每个公众号建立独立配置
- 在中转服务中增加审计日志和重试队列

### 15.3 动态公网 IP 用户

- 优先使用固定公网服务器 Webhook
- 或在公网 IP 变化后及时更新微信白名单
- 不建议为了方便而长期使用不受控的代理出口
