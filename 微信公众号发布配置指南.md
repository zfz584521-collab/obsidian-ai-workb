# AI Workbench 微信公众号发布配置指南

更新时间：2026-06-16

这份文档用于以后重新接入或排查微信公众号发布。当前插件支持把 Obsidian 当前 Markdown 笔记提交到微信公众号草稿箱，提交后仍需要到公众号后台预览、检查排版，并由你手动群发或发布。

> 注意：微信公众号后台的“开发接口管理”入口已经开始迁移。旧入口仍可能显示在“设置与开发”下面，但页面会提示迁移到微信开发者平台。后续获取 AppID、AppSecret、配置 IP 白名单和查看接口权限时，以微信开发者平台或公众号后台当前显示为准。

---

## 1. 接入方式怎么选

AI Workbench 对微信公众号提供两种方式：

| 方式 | 适合情况 | 说明 |
|---|---|---|
| 官方 API | 个人使用、只有一个公众号、能配置公网 IP 白名单 | 插件直接调用微信接口创建草稿，是当前推荐方式 |
| Webhook | 多账号、团队协作、希望把密钥放服务器、需要更复杂排版 | 插件把内容发给你的中转服务，由服务端再调用微信接口 |

目前我们使用的是 **官方 API**。

官方 API 会做这些事：

1. 用 AppID 和 AppSecret 获取 `access_token`。
2. 上传封面图为微信永久素材。
3. 上传正文里的本地图片。
4. 把 Markdown 转成适合公众号阅读的 HTML。
5. 调用微信“新增草稿”接口。
6. 返回草稿 ID。

不会做这些事：

- 不会自动群发。
- 不会公开发布。
- 不会绕过公众号后台审核。
- 不会在官方 API 失败后自动改用 Webhook。

---

## 2. 准备条件

接入前先确认：

- 你能登录 [微信公众平台](https://mp.weixin.qq.com/)。
- 你是公众号管理员，或有开发配置权限。
- 公众号能查看 AppID。
- 公众号能生成或重置 AppSecret。
- 公众号有素材管理、草稿箱相关接口权限。
- 当前电脑访问微信接口的公网出口 IP 可以加入白名单。
- Obsidian 所在电脑可以访问 `https://api.weixin.qq.com`。

如果是公司网络、代理、VPN、热点或家庭宽带，公网出口 IP 可能变化。后面如果突然不能发布，优先查 IP 白名单。

---

## 3. 获取 AppID 和 AppSecret

### 3.1 新版入口

现在优先按这个路径找：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)。
2. 左侧进入 `设置与开发`。
3. 点击 `开发接口管理`。
4. 如果页面提示“开发管理模块迁移通知”，按提示进入微信开发者平台。
5. 在微信开发者平台中找到当前公众号对应的开发配置或接口管理页面。
6. 找到 `AppID` 和 `AppSecret`。

如果新平台页面名称有变化，记住目标不是“公众号原始 ID”，而是：

- `AppID`：通常以 `wx` 开头，例如 `wx123...`。
- `AppSecret`：一串较长的密钥字符串。

### 3.2 旧版入口

如果你的后台还保留旧页面，可以按这个路径找：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)。
2. 进入 `设置与开发`。
3. 进入 `基本配置` 或 `开发接口管理`。
4. 找到 `开发者 ID(AppID)`。
5. 找到 `开发者密码(AppSecret)`。

### 3.3 AppSecret 看不到怎么办

很多情况下后台不会再次明文显示旧 AppSecret。处理方式：

1. 点击重置或生成 AppSecret。
2. 按公众号后台要求完成管理员扫码或身份验证。
3. 复制新的 AppSecret。
4. 立刻更新 AI Workbench 设置。

注意：

- AppSecret 重置后，旧密钥立即失效。
- 不要把 AppSecret 发到聊天、截图、Git、网盘共享文档里。
- 不要把公众号原始 ID、微信号、二维码信息填成 AppID。

## 4. 配置 IP 白名单

微信获取 `access_token` 时会校验请求来源 IP。AI Workbench 是在本机 Obsidian 里直接请求微信接口，所以要把当前网络的公网出口 IP 加到公众号白名单。

### 4.1 查询公网 IP

在浏览器里搜索：

```text
我的 IP
```

或访问常用 IP 查询网站，复制显示出来的 IPv4 地址。

不要填写这些地址：

- `127.0.0.1`
- `localhost`
- `192.168.x.x`
- `10.x.x.x`
- `172.16.x.x` 到 `172.31.x.x`

这些都是本地或内网地址，微信服务器看不到。

### 4.2 添加白名单

新版入口大致是：

1. 进入微信开发者平台。
2. 找到当前公众号。
3. 进入开发配置、接口管理或安全配置。
4. 找到 `IP 白名单`。
5. 添加当前公网 IPv4。
6. 保存。

旧版入口大致是：

1. 微信公众平台。
2. `设置与开发`。
3. `基本配置` 或 `开发接口管理`。
4. 找到 `IP 白名单`。
5. 添加当前公网 IPv4。
6. 保存。

### 4.3 常见 IP 问题

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 测试连接失败，提示认证失败 | IP 未加入白名单 | 查询当前公网 IP 后重新加入 |
| 昨天能用，今天不能用 | 家庭宽带或热点 IP 变了 | 更新白名单 |
| 开了 VPN 后失败 | 微信看到的是 VPN 出口 IP | 关闭 VPN 或把 VPN 出口 IP 加白名单 |
| 公司网络失败 | 公司统一出口 IP 与本机查询不同 | 让网络管理员确认出口 IP |

## 5. 在 AI Workbench 配置微信公众号

### 5.1 打开配置界面

在 Obsidian 中进入：

```text
设置 → 第三方插件 → AI Workbench → 发布平台
```

找到 `微信公众号`。

### 5.2 基础配置

按顺序操作：

1. 打开 `微信公众号` 右侧开关。
2. 接入方式选择 `官方 API`。
3. 点击 `配置`。
4. 确认 `启用平台` 已打开。
5. 根据需要打开 `工作台默认选中`。
6. 填写 `AppID`。
7. 填写 `AppSecret`。
8. 可选填写 `默认作者`。
9. 保存设置。

字段说明：

| 字段 | 必填 | 说明 |
|---|---:|---|
| AppID | 是 | 微信公众号开发者 ID，通常以 `wx` 开头 |
| AppSecret | 是 | 微信公众号开发者密码 |
| 默认作者 | 否 | 创建草稿时写入公众号作者字段 |
| 工作台默认选中 | 否 | 打开工作台时自动勾选微信公众号 |

### 5.3 请求超时建议

发布平台区域有 `发布请求超时`。

建议：

- 纯文字文章：60 秒。
- 有 1-3 张图片：90 秒。
- 有多张大图：120 秒。
- 网络较慢：120-180 秒。

如果图片上传到一半失败，先把超时调到 120 秒。

## 6. 测试连接

在微信公众号设置行点击 `测试连接`。

测试连接会做：

1. 读取已保存的 AppID 和 AppSecret。
2. 请求微信 `access_token` 接口。
3. 验证网络、密钥和 IP 白名单是否基本可用。

测试连接不会做：

- 不会读取当前笔记。
- 不会上传图片。
- 不会创建草稿。
- 不会公开发布。

如果提示连接成功，说明基础认证通过。素材上传和草稿创建仍要通过第一次实际发布验证。

## 7. 准备公众号文章

### 7.1 推荐 Markdown 结构

```markdown
---
title: 文章标题
summary: 文章摘要
tags:
  - AI
  - 效率工具
---

# 文章标题

开头第一段，建议不要太长。

## 第一部分标题

正文段落。每段尽量短一些。

![[images/cover.png]]

*图片说明，可以作为公众号里的图注。*
```

### 7.2 标题读取顺序

插件按这个顺序读取标题：

1. Frontmatter 里的 `title`。
2. 正文第一个一级标题。
3. Markdown 文件名。

发布编辑器打开后可以手动改标题。

### 7.3 图片要求

支持这些图片写法：

```markdown
![[images/example.png]]
```

```markdown
![图片说明](images/example.png)
```

```markdown
![图片说明](https://example.com/example.png)
```

建议：

- 封面图优先使用本地图片。
- 正文关键图片优先使用本地图片，让插件上传到微信。
- 发布前确认图片在 Obsidian 里能正常显示。
- 文件名尽量不要包含特殊符号。

## 8. 从工作台发布到草稿箱

### 8.1 操作步骤

1. 打开要发布的 Markdown 笔记。
2. 打开 AI Workbench 侧边栏。
3. 找到 `发布到草稿箱`。
4. 勾选 `微信公众号`。
5. 点击 `编辑并发布`。
6. 在弹窗左侧检查统一内容：
   - 标题
   - 正文
   - 摘要
   - 标签
7. 在右侧检查封面与媒体：
   - 第一张图通常会自动作为封面候选。
   - 也可以手动点 `设为封面`。
   - 公众号官方 API 必须有封面。
8. 如果只想给公众号单独改标题、正文或摘要，切换到平台设置里修改覆盖内容。
9. 点击 `发布 1 个草稿`。
10. 等待结果区显示成功。
11. 复制或记录草稿 ID。
12. 登录公众号后台，进入草稿箱检查。

### 8.2 命令面板发布

也可以打开 Obsidian 命令面板，执行：

```text
发布当前笔记到平台草稿箱
```

如果已配置默认平台，插件会使用默认平台；否则会使用已启用并配置完成的平台。

### 8.3 发布成功后去哪里看

进入公众号后台：

```text
内容与互动 → 草稿箱
```

找到刚创建的草稿，检查：

- 标题是否正确。
- 作者是否正确。
- 封面是否正确。
- 图片是否全部显示。
- 段落间距是否舒服。
- 小标题样式是否正常。
- 手机预览是否正常。
- 链接是否有效。

## 9. 当前公众号排版规则

插件现在会把 Markdown 转成更适合公众号阅读的 HTML：

- 段落之间增加空行。
- 段落行高加大，提升手机阅读体验。
- 长段落会尽量拆短。
- Markdown 标题会变成公众号风格的小节标题。
- 单独一行的加粗短句也会识别为小节标题。
- 小节标题前会带编号，例如 `01`、`02`。
- 小节标题有浅色背景和左侧强调线。
- 图片居中显示。
- 图片后面的斜体短句会作为图注处理。

推荐写法：

```markdown
## 为什么这种用法突然火了？

这里写正文。正文建议每段控制在 3-5 行以内。

**日常三步走，让知识库自己长大**

这里继续写正文。
```

不推荐：

- 整篇文章只有大段正文，没有标题。
- 一个段落写十几行。
- 大量复杂表格。
- 大量代码块。
- 依赖 Obsidian callout、Mermaid、数学公式等特殊语法。

## 10. 常见问题与处理

### 10.1 设置里开启了，工作台还是不能勾选

处理顺序：

1. 确认已点击保存。
2. 确认微信公众号是 `启用平台`。
3. 确认接入方式是 `官方 API`。
4. 确认 AppID 和 AppSecret 都已填写。
5. 关闭再打开 AI Workbench 侧边栏。
6. 必要时重启 Obsidian。

新版插件已经会在保存设置后刷新工作台。如果仍然不可选，通常是字段没填完整或配置没有保存。

### 10.2 提示“微信公众号 未启用”

常见原因：

- 只打开了平台总开关，没有进入配置弹窗打开 `启用平台`。
- 没有保存设置。
- 当前平台配置不完整。

处理：

1. 进入 `设置 → 第三方插件 → AI Workbench → 发布平台`。
2. 打开微信公众号配置。
3. 确认启用平台、AppID、AppSecret。
4. 保存。
5. 回到工作台重新打开发布区。

### 10.3 测试连接失败

按顺序排查：

1. AppID 是否以 `wx` 开头。
2. AppSecret 是否复制完整。
3. AppID 和 AppSecret 是否属于同一个公众号。
4. AppSecret 是否刚被重置。
5. 当前公网 IP 是否加入白名单。
6. 是否开了 VPN 或代理。
7. Obsidian 是否能联网。
8. 公众号接口权限是否正常。

如果错误类似 IP 不在白名单，直接更新 IP 白名单。

### 10.4 发布失败：无法连接微信公众号接口

可能原因：

- 网络不通。
- 微信接口临时不可用。
- 代理或防火墙拦截 Obsidian。
- 请求超时太短。

处理：

1. 浏览器打开 `https://api.weixin.qq.com` 看是否能访问。
2. 临时关闭代理或 VPN。
3. 把发布请求超时调到 120 秒。
4. 重试。

### 10.5 发布失败：封面相关错误

公众号官方草稿必须有封面。

处理：

1. 在发布弹窗右侧确认有图片。
2. 点击某张图片的 `设为封面`。
3. 确认图片文件没有被移动或删除。
4. 确认图片格式是常见格式，例如 PNG、JPG。
5. 图片过大时先压缩。

### 10.6 发布失败：正文图片上传失败

常见原因：

- 图片路径不存在。
- 图片在 Obsidian 里已经显示失败。
- 图片文件太大。
- 图片格式不受微信当前接口支持。
- 网络上传中断。
- 请求超时太短。

处理：

1. 在 Obsidian 中确认图片能正常预览。
2. 把图片放到当前库内的稳定路径。
3. 压缩大图。
4. 把发布请求超时调到 120 秒或更高。
5. 重新发布。

### 10.7 发布成功，但公众号后台找不到草稿

排查：

1. 登录的公众号是否和 AppID 对应。
2. 是否进入了正确的 `内容与互动 → 草稿箱`。
3. 是否用管理员管理多个公众号。
4. 发布结果里的草稿 ID 是否存在。
5. 是否切到了其他账号或其他主体。

### 10.8 发布成功，但排版不满意

处理建议：

1. 回到 Markdown，把长段落拆短。
2. 给每一部分加 `## 小标题`。
3. 图片下方用斜体写图注。
4. 避免复杂表格和代码块。
5. 重新发布一个草稿。
6. 在公众号后台做最后微调。

### 10.9 草稿创建失败

如果认证和图片上传都成功，但最后创建草稿失败，可能是：

- 公众号没有草稿箱接口权限。
- 标题或正文包含微信不接受的内容。
- 封面素材 ID 无效。
- 正文 HTML 有微信不支持的标签。
- 公众号素材数量、草稿数量或接口调用次数达到限制。
- 微信接口临时异常。

处理：

1. 简化文章，只保留标题、几段文字和一张封面测试。
2. 如果简单文章能成功，再逐步加回图片和复杂内容。
3. 检查公众号接口权限。
4. 删除不需要的旧草稿或素材。
5. 稍后重试。

- [ ] 微信公众号平台已启用。
- [ ] 接入方式选择 `官方 API`。
- [ ] AppID 已填写。
- [ ] AppSecret 已填写。
- [ ] 当前公网 IP 已加入白名单。
- [ ] 测试连接成功。
- [ ] 当前打开的是 Markdown 笔记。
- [ ] 标题不为空。
- [ ] 正文段落已拆短。
- [ ] 每个主题有小标题。
- [ ] 图片在 Obsidian 中能正常显示。
- [ ] 已选择封面。
- [ ] 发布成功后已进入公众号后台预览。
- [ ] 正式群发前已检查手机预览。

---

## 12. 安全注意事项

- 不要把 `data.json` 发给别人。
- 不要把 AppSecret 提交到 Git。
- 不要在截图里露出 AppSecret。
- 不要把 AppSecret 写进文章正文。
- 怀疑泄露时立即重置 AppSecret。
- 如果多人协作，优先考虑 Webhook，把 AppSecret 放在服务器端。
