# 右侧栏自定义动作分类视觉设计

## 目标

将右侧 AI Workbench 中的 25 个自定义动作按现有类别清晰分区，降低长列表的扫描成本，同时保持 Obsidian 原生、克制的视觉风格。

## 选定方向

采用视觉方案 B：克制卡片分区。

- 每个类别使用独立的浅底卡片区块。
- 卡片左侧使用 3px 分类色条。
- 类别标题使用与色条对应的低饱和文字色。
- 标题右侧显示该类别的动作数量。
- 动作按钮保持统一中性色，不为每个动作单独上色。
- 深浅主题均使用 Obsidian CSS 变量作为底色、边框和按钮颜色来源。

## 分类结构

沿用 `PRESET_CATEGORIES` 中的现有 7 类及当前顺序：

1. 基础处理：5 个
2. 小红书：3 个
3. 短视频：5 个
4. 公众号：5 个
5. 翻译：2 个
6. 代码相关：2 个
7. 其他：3 个

没有类别的自定义动作继续进入“未分类”区块。

## 视觉规则

### 类别卡片

- 圆角使用现有侧栏组件的圆角尺度。
- 背景使用 `var(--background-secondary)`。
- 边框使用 `var(--background-modifier-border)`。
- 类别之间保留明确的垂直间距。
- 卡片内部包含类别标题和双列动作按钮。

### 分类颜色

颜色只承担定位作用，不作为大面积背景：

- 基础处理：蓝色
- 小红书：玫红色
- 短视频：紫色
- 公众号：绿色
- 翻译：青色
- 代码相关：橙色
- 其他及未分类：中性灰色

通过类别 CSS class 设置色条和标题颜色。浅色主题下使用较深色值，深色主题下使用较亮色值，保证对比度。

### 动作按钮

- 继续使用双列网格。
- 所有自定义动作按钮保持相同中性色底纹。
- 去掉当前自定义按钮的虚线边框，避免 25 个按钮产生过多视觉噪声。
- Hover 状态继续使用 Obsidian 的交互高亮色。
- 长名称允许自然换行，卡片内按钮高度保持一致。

## 实现范围

修改范围保持在右侧栏渲染与样式：

- `main.ts`：为类别容器增加稳定的类别 class，并显示动作数量。
- `styles.css`：实现卡片、色条、分类标题颜色以及统一按钮样式。

不修改 Prompt 内容、执行逻辑、配置格式、导入导出和快捷键。

## 状态与兼容

- 空类别不渲染，延续当前行为。
- 未知类别仍可渲染，并回退为中性灰色。
- 现有用户的 `data.json` 无需迁移。
- 深色和浅色 Obsidian 主题均应可读。

## 验证

1. 运行 `npm run build`，确认 TypeScript 和打包成功。
2. 在包含全部 25 个动作的侧栏中确认 7 个分类顺序正确。
3. 确认每个分类数量正确。
4. 分别在 Obsidian 深色与浅色主题查看类别边界、标题对比度和按钮 Hover。
5. 点击各分类中的动作，确认执行行为没有变化。
